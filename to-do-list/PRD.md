# 오늘의 할일(To-Do Today) 웹페이지 PRD

## 목적
사용자가 그날의 할 일을 빠르게 등록, 확인, 완료 처리할 수 있는 간단하고 직관적인 웹 인터페이스를 제공한다. 하루 단위 집중 작업을 돕는 가벼운 생산성 도구를 목표로 한다.

## 주요 사용자(페르소나)
- 바쁜 직장인: 당일 우선순위만 간단히 관리
- 학생: 오늘의 과제/공부할 항목 관리
- 프리랜서: 당일 작업 항목 체크 및 타임블록용

## 핵심 기능 (MVP)
- 할 일 추가: 제목(필수), 메모(선택), 우선순위(낮음/보통/높음)
- 오늘 목록 보기: 등록된 항목을 리스트로 표시
- 완료 표시: 체크하면 완료 처리 및 목록에서 시각적 분리
- 삭제/수정: 항목 편집 및 삭제
- 간단한 필터: 전체/미완료/완료

## 사용자 흐름
1. 페이지 진입 → 오늘 날짜 자동 표시
2. 입력창에 제목 입력 후 추가 버튼 클릭
3. 목록에서 체크박스로 완료 토글, 항목 클릭 시 편집 모달
4. 필터로 미완료 항목만 표시 가능

## UI/UX 요구사항
- 반응형: 모바일 우선 디자인
- 직관적 인터페이스(단일 화면으로 모든 동작 가능)
- 접근성 고려(키보드로 빠른 추가/탭 이동)

## 비기능 요구사항
- 로컬 우선 저장: 브라우저 로컬스토리지 사용(MVP), 이후 서버 동기화 옵션
- 빠른 로드 및 저용량: 단일 HTML/CSS/JS 파일 또는 경량 프레임워크
- 보안: 저장 데이터는 클라이언트에 국한(민감정보 비저장)

## 성공 기준
- 한 번 보고도 사용법을 이해할 수 있음
- 항목 추가 후 2초 이내 목록에 반영
- 모바일/데스크톱에서 모두 사용 가능

## 향후 확장 아이디어
- 날짜별 이력 보기
- 알림/리마인더(브라우저 알림)
- 로그인 및 서버 동기화

## MVP 구현 범위
- 프론트엔드 단일 페이지(HTML/CSS/Vanilla JS)
- 로컬스토리지 기반 저장
